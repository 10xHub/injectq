    üî¨ Understanding the Results¶

    Time Units¶

    • ns (nanosecond) = 0.000001 milliseconds
    • Œºs (microsecond) = 0.001 milliseconds
    • ms (millisecond) = 0.001 seconds

    Operations Per Second (OPS)¶

    • Higher is better
    • Calculated as 1 / mean_time
    • Shows throughput capacity

    Outliers¶

    • 1.5 IQR: Moderate outliers (expected)
    • 1 StdDev: Significant outliers (investigate if many)

    Comparison Operators¶

    • (1.0) = Baseline (fastest test)
    • (2.5) = 2.5√ó slower than baseline
    • (>1000.0) = More than 1000√ó slower

    üí° Performance Tips¶

    ‚úÖ DO¶

    • Use singleton scope for shared services (1 Œºs cached)
    • Use factory bindings for lazy init (271 ns)
    • Batch service registrations at startup (779 Œºs for 100)
    • Check with has_service before complex ops (272 ns)

    ‚ö†Ô∏è CONSIDER¶

    • Transient scope adds overhead (4.7 Œºs vs 1 Œºs)
    • Thread-safe container is 24√ó slower (still fast at 24.7 Œºs)
    • Deep dependency trees perform well (1 Œºs for 5 levels)

    ‚ùå AVOID¶

    • Creating containers in hot paths (24.3 Œºs each)
    • Overusing transient scope for shared services
    • Unnecessary thread-safe containers (use only when needed)
